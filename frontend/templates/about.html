Technical Architecture & Institutional Overview

Comprehensive system documentation for the StressIntel PRO diagnostic suite.

1. About the Project

StressIntel PRO is a research-grade institutional diagnostic tool designed to analyze multi-factorial stress patterns in academic cohorts. By integrating behavioral biomarkers, environmental conditions, and physiological markers, the system provides a holistic stress risk assessment powered by high-performance machine learning and generative AI.

2. Model Zoo (Multi-Model Architecture)

Primary

XGBoost Classifier

Handles the 21-parameter stress prediction. Optimized for tabular data with high precision across Low, Medium, and High stress classes.

Generative

Ollama Llama 3

Provides deep reasoning and clinical summaries. Synthesizes complex data into human-readable institutional reports.

Heuristic

Research Intelligence Model

A deterministic expert system providing fail-safe diagnostic reasoning when local LLM resources are constrained.

Visual

Visual Intelligence Engine

Computer vision classifier (Haar + Heatmap) for detecting facial stress markers and generating impactful report visualizations.

8. System Workflow

sequenceDiagram
    participant U as User
    participant F as Frontend
    participant B as Backend
    participant ML as ML Service
    participant LLM as Local LLM
    
    U->>F: Submits Research Data
    F->>F: Capture Diagnostic Frame
    F->>B: POST /api/research/analyze
    B->>ML: Run XGBoost & Visual Classifier
    ML-->>B: Prediction & Annotated Frame
    B->>LLM: Request Deep Insight
    alt LLM Online
        LLM-->>B: Llama 3 Reasoning

        B->>B: Run Heuristic Fallback
    end
    B-->>F: Return Combined Diagnostic

9. Technical Requirements

  • Runtime: Python 3.10+ / Flask 3.0
  • Local LLM: Ollama (Llama 3 8B recommended)
  • Memory: 16GB RAM (Minimum 8GB for Heuristic Mode)
  • Storage: 5GB (Model weights + Llama 3)
  • Browser: WebRTC compatible (Chrome/Edge/Firefox)

Future Possibilities

Integration of real-time wearable data (HRV/ECG), Federated Learning for cross-institutional cohorts without data sharing, and VR-based stress mitigation modules.
